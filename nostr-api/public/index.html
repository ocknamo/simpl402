⚡ x402 over Lightning Network - Nostr API

This is a REST API implementation that requires Lightning Network payment to access protected resources. It follows the x402 protocol specification.

🔌 API Endpoint

GET /nostr/secret-key

Returns a Nostr secret key after Lightning payment verification.

🔄 How It Works

Step 1: Initial Request 402 Payment Required

curl -i http://localhost:8787/nostr/secret-key

Response includes a PAYMENT-REQUIRED header with a Base64-encoded Lightning invoice:

HTTP/1.1 402 Payment Required
PAYMENT-REQUIRED: eyJzY2hlbWUiOiJsaWdodG5pbmciLCJuZXR3b3JrIjoiYml0Y29pbiIsImludm9pY2UiOiJsbmJjMS4uLiJ9

Step 2: Decode the Invoice

echo "eyJ..." | base64 -d | jq

Returns:

{
  "scheme": "lightning",
  "network": "bitcoin",
  "invoice": "lnbc1u1p5..."
}

Step 3: Pay the Invoice

Use your Lightning wallet to pay the invoice (100 sats).

Step 4: Access with Payment Signature 200 OK

curl -i http://localhost:8787/nostr/secret-key \
  -H "PAYMENT-SIGNATURE: eyJ..."

Response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "secretKey": "nsec1xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
}

🔒 Security Features

  • One-time use: Each invoice can only be used once
  • Expiration: Invoices expire after 1 hour
  • Payment verification: Verified through coinos.io Lightning node

🛠️ Technology Stack

  • Cloudflare Workers (Serverless)
  • Lightning Network (coinos.io API)
  • Cloudflare KV (Invoice tracking)
  • TypeScript

⚡ x402 over Lightning Network - Nostr API

Lightning Network による支払いが必要な REST API の実装です。 x402 プロトコル仕様に準拠しています。

🔌 API エンドポイント

GET /nostr/secret-key

Lightning 支払い検証後に Nostr の秘密鍵を返します。

🔄 動作の流れ

ステップ 1: 初回リクエスト 402 Payment Required

curl -i http://localhost:8787/nostr/secret-key

レスポンスには Base64 エンコードされた Lightning invoice を含む PAYMENT-REQUIRED ヘッダーが含まれます：

HTTP/1.1 402 Payment Required
PAYMENT-REQUIRED: eyJzY2hlbWUiOiJsaWdodG5pbmciLCJuZXR3b3JrIjoiYml0Y29pbiIsImludm9pY2UiOiJsbmJjMS4uLiJ9

ステップ 2: Invoice のデコード

echo "eyJ..." | base64 -d | jq

結果：

{
  "scheme": "lightning",
  "network": "bitcoin",
  "invoice": "lnbc1u1p5..."
}

ステップ 3: Invoice の支払い

Lightning ウォレットを使用して invoice を支払います（100 sats）。

ステップ 4: 支払い署名でアクセス 200 OK

curl -i http://localhost:8787/nostr/secret-key \
  -H "PAYMENT-SIGNATURE: eyJ..."

レスポンス：

HTTP/1.1 200 OK
Content-Type: application/json

{
  "secretKey": "nsec1xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
}

🔒 セキュリティ機能

  • 一度のみ使用可能: 各 invoice は一度しか使用できません
  • 有効期限: Invoice は 1 時間後に期限切れになります
  • 支払い検証: coinos.io Lightning ノードで検証されます
